# 한경 산업별 시장 대시보드

한국경제 데이터센터의 FACTSET 업종 분류·종목 등락 정보를 읽고, 다음 종목군만 남겨 GitHub Pages에서 시각화합니다.

- 코스피: KOSPI 200 구성종목
- 코스닥: 시가총액 상위 100종목
- 업종 등락률: 포함 종목의 시가총액 가중 평균
- 자동 갱신: 평일 한국시간 16:05 전후
- 기본 화면: 상승률 상위 5개 업종 + 하락률 하위 5개 업종
- 업종 클릭: 해당 업종에 포함된 대상 기업 전체 펼침
- 검색: 기본 TOP 10 밖의 업종·종목도 전체 수집 데이터에서 검색

## 이번 디자인 예시

처음 파일의 예시 데이터는 5개 업종·12개 기업뿐이라 실제 300종목 대시보드의 밀도가 보이지 않았습니다. 수정본은 다음처럼 구성했습니다.

- 표시 업종 10개: 상승 TOP 5 + 하락 TOP 5
- 디자인 예시 세부 기업 86개
- 화면 상단의 대상 종목은 실제 운영 기준 300개로 표시
- GitHub Actions가 정상 실행되면 예시 JSON은 실제 장 마감 데이터로 교체
- 실제 매칭 종목이 250개 미만이면 기존 정상 데이터를 덮어쓰지 않고 작업 실패

## 파일 구조

```text

scripts/diagnose_hankyung.py
```

## GitHub에 올리는 순서

1. 새 저장소를 만들고 이 폴더의 파일을 그대로 업로드합니다.
2. 저장소의 `Settings → Pages`로 이동합니다.
3. `Build and deployment`에서 `Deploy from a branch`를 선택합니다.
4. Branch를 `main`, 폴더를 `/(root)`로 지정하고 저장합니다.
5. `Actions → 산업별 시장 대시보드 갱신 → Run workflow`를 한 번 실행합니다.
6. 실행이 끝난 뒤 Pages 주소를 Notion에서 `/embed`로 붙입니다.

## 자동 갱신 시각

GitHub Actions cron은 UTC 기준이라 `5 7 * * 1-5`로 설정했습니다. 이는 평일 한국시간 16:05입니다. GitHub 사정에 따라 실제 시작은 몇 분 늦을 수 있지만, 생성되는 데이터는 최근 장 마감 거래일 기준입니다.

## 화면 동작

- `통합`: KOSPI 200과 KOSDAQ 시총 상위 100을 합쳐 업종 수익률 계산
- `코스피200`: KOSPI 200 구성기업만으로 업종 수익률 재계산
- `코스닥100`: KOSDAQ 시총 상위 100기업만으로 업종 수익률 재계산
- 기본 화면: 각 시장 필터에서 상승 TOP 5와 하락 TOP 5를 다시 선정
- 검색: 기업명·업종명·6자리 종목코드 검색
- 업종 카드: 클릭하면 기업명, 시장, 현재가, 등락률, 시가총액 표시

## 오류가 날 때

한경이 페이지 구조/API를 변경하면 수집 결과가 줄어들 수 있습니다. 정상 데이터가 덮어써지는 것을 막기 위해 250종목 미만으로 매칭되면 작업을 실패시킵니다.

로컬 또는 Actions에서 아래 진단 스크립트를 실행하면 `debug/summary.json`이 생성됩니다.

```bash
python scripts/diagnose_hankyung.py
```

## 데이터 이용 주의

개인용 Notion 대시보드 범위에서 사용하고, 공개 재배포·상업적 사용 전에는 한국경제와 데이터 제공자의 이용 조건을 확인하세요.

## 화면 구성 변경

- 상단 제목과 4개 시장 요약 카드는 제거했습니다.
- 페이지는 시장 필터와 검색창부터 바로 시작합니다.
- 장 마감 기준일과 마지막 갱신 시각은 하단 출처 영역에서 확인할 수 있습니다.
